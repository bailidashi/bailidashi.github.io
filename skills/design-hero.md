当用户输入 /design-hero 或想要优化网站外观时：

## 第一步：判断改版类型

先问清楚用户意图：

> "你想要**大改版**（换配色/布局/结构）还是**小打磨**（加动效/质感/细节）？"

### 大改版模式

1. 先读 index.html，了解现有内容
2. 先问用户选**设计系统**：

> "你想要哪种设计风格？**小清新**（浅色双色交替 + 波浪过渡）还是**南孚科技风**（深蓝主色 + 亮蓝点缀 + 光晕背景）？"

3. 确认设计系统后，再在对应配色方案里选具体颜色
4. **生成独立预览文件**，不碰原文件
5. 用户迭代确认后，再合入 index.html

### 设计系统

#### 系统 A：小清新
- 全页浅色为主，仅末尾 section 用深底压轴
- 双色交替 section（奶杏→浅色→奶杏→浅色），形成清新节奏
- section 之间用 SVG 海浪曲线过渡
- Hero 左右斜切分屏 + 波浪装饰
- 卡片带渐变遮罩，柔和阴影

#### 系统 B：南孚科技风
- 深蓝 `#0A2B4E` 主色 + 亮蓝 `#3B82F6` 点缀
- 白色/浅灰 section 交替
- 全局蓝色光晕背景（radial-gradient + blur）
- Hero 左右布局 + 大数字统计块（南孚风核心）
- 渐变标题（navy→blue background-clip: text）
- 导航半透明 navy + backdrop-filter 模糊
- 卡片 hover 阴影带蓝色调
- 按钮圆角饱满、硬朗悬浮感

### 小打磨模式

给出 4-5 个可选的视觉增强方向：

| 选项 | 效果 | 改动量 |
|------|------|--------|
| 过渡光斑 | section 间雾状薄荷光晕，柔和过渡 | 少量 CSS + 几个 div |
| 3D 卡片倾斜 | 卡片 hover 时微旋转 + 景深阴影 | 改 2 行 CSS |
| 头像光环旋转 | 头像外围渐变色圆环缓慢旋转 | 加伪元素 + 动画 |
| 鼠标跟随光晕 | 页面跟随鼠标移动的柔光 | 少量 CSS + ~10 行 JS |
| 文字渐入动效 | 标题或标语逐个字淡入 | 少量 CSS + JS |

用户选好后 → 生成预览 → 确认 → 合入。

## 核心原则

- **无论如何都不要在没有预览的情况下直接改原文件**
- 一次只改一个方向，让用户看清差异再继续
- 保持现有配色和字体风格，不做破坏性变更
- 移动端自动降级（动效在手机上隐藏或简化）
- 新增的视觉效果用纯 CSS 实现，不引入图片或三方库
- section 过渡区域是视觉优化的重点（用户的视觉注意力最集中在那里）
